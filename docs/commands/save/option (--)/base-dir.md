# --base-dir

Base directory (auto-fills all directories from standard structure)

## Synopsis

```bash
cupertino save --base-dir <path>
```

## Description

Specifies the base directory containing the standard Cupertino directory structure. When provided, automatically derives paths for all subdirectories:

- `docs/` - Apple Developer documentation
- `swift-evolution/` - Swift Evolution proposals
- `swift-org/` - Swift.org documentation
- `packages/` - Swift package READMEs
- `metadata.json` - Crawl metadata
- `apple-documentation.db` - Apple docs database (plus the sibling per-source DBs)

Individual directory options (`--docs-dir`, `--evolution-dir`, etc.) override the base-derived paths.

## Default

`~/.cupertino`

## Examples

### Index Default Base Directory
```bash
cupertino save --all
```

Equivalent to:
```bash
cupertino save --base-dir ~/.cupertino
```

### Index Custom Base Directory
```bash
cupertino save --base-dir ./my-cupertino
```

Auto-fills:
- `--docs-dir ./my-cupertino/docs`
- `--evolution-dir ./my-cupertino/swift-evolution`
- `--swift-org-dir ./my-cupertino/swift-org`
- `--packages-dir ./my-cupertino/packages`
- `--metadata-file ./my-cupertino/metadata.json`

### Absolute Path
```bash
cupertino save --base-dir /Users/username/Documents/cupertino
```

## Expected Structure

The base directory should follow this layout:

```
base-dir/
├── docs/                    # Apple docs (from fetch --source apple-docs)
│   ├── metadata.json
│   ├── swiftui/
│   └── foundation/
├── swift-evolution/         # Proposals (from fetch --source swift-evolution)
│   ├── SE-0001-*.md
│   └── SE-0002-*.md
├── swift-org/               # Swift.org docs (from fetch --source swift-org)
│   └── *.md
├── packages/                # Package READMEs (from fetch --source packages)
│   └── owner/
│       └── repo/
└── apple-documentation.db   # Generated by save (+ sibling per-source DBs)
```

## Notes

- Simplifies command invocation when using standard structure
- Tilde (`~`) expansion is supported
- Individual options take precedence over base-derived paths
- Base directory does not need to exist yet (subdirectories created as needed)
- Recommended approach for most users
- Useful for managing multiple documentation sets
